# 📄 Требования к системе «Сайт по продаже металла» (Версия 1.0)

## 🏗 1. Архитектура и инфраструктура

| Компонент | Технология / Решение |
|-----------|---------------------|
| Фронтенд (клиент) | Vue 3 |
| Фронтенд (админка) | Vue 3 |
| Бэкенд | Go (Golang) |
| Веб-сервер | Nginx |
| ORM | GORM |
| БД (Dev) | SQLite |
| БД (Prod) | PostgreSQL |
| Контейнеризация | Docker Compose (2 файла) |
| Конфигурация | Переменные окружения (`.env`) |
| Хранение файлов | Локальный диск сервера (без S3) |

### Структура репозитория (монорепозиторий)
```text
/
├── client/              # Vue 3 — клиентская часть (публичный сайт)
├── admin/               # Vue 3 — панель администратора
├── backend/             # Go — API, бизнес-логика, работа с БД
├── nginx/               # Конфигурационные файлы Nginx
├── storage/             # Локальное хранилище загруженных файлов
├── docker-compose.yml   # Окружение разработки
├── docker-compose.prod.yml # Продуктивное окружение (сервисы с префиксом .prod)
├── .env                 # Переменные окружения (не коммитить в git)
└── README.md
```

### Требования к развертыванию
- Файлы Docker Compose расположены в корне проекта.
- Продуктивные сервисы имеют префикс `.prod` для изоляции от дев-окружения.
- Nginx настроен на проксирование запросов к бэкенду и отдачу статики (включая загруженные изображения из `storage/`).
- Различия между окружениями реализуются **только** через Docker Compose и переменные окружения, без изменения конфигов сервисов.

## 🌐 2. Клиентская часть (Публичный сайт)

### Страницы
1. **Главная**
   - Приветственный баннер.
   - Галерея изображений продукции.
   - Редактируемые контентные блоки (Markdown).

2. **Каталог**
   - Отображение до 8000 единиц продукции.
   - Навигация по категориям и подкатегориям.
   - Полнотекстовый поиск по всем полям товара (название, описание, артикул, характеристики).
   - Фильтрация и пагинация результатов.
   - Карточка товара: изображение, описание, характеристики, кнопка «Связаться».

3. **Контакты**
   - Описание компании.
   - Контактные телефоны и адреса электронной почты.
   - Интерактивная карта с местоположением (сервис карт на усмотрение разработки).
   - Редактируемые контентные блоки (Markdown).

### Форма обратной связи (на всех страницах)
- Поля: `Имя*`, `Телефон*`, `Тема` (необязательно).
- Валидация: маска телефона, проверка формата email (если будет добавлен).
- Защита: базовая защита от спама (honeypot или капча).
- Юридическое: чекбокс «Согласен на обработку персональных данных» со ссылкой на политику конфиденциальности.
- Обработка: заявка сохраняется в БД, отображается в админке.

> ⚠️ **Важно**: Сайт не содержит функционала корзины, оформления заказа или онлайн-оплаты. Модель взаимодействия: «Просмотр каталога → Заявка → Менеджер связывается с клиентом».

## ⚙️ 3. Административная панель

### Авторизация и роли
- Аутентификация по логину и паролю.
- Пароли хранятся в хэшированном виде (bcrypt/argon2).
- Система ролей (RBAC):

| Роль | Права доступа |
|------|--------------|
| **Администратор** | Полный доступ ко всем функциям, управление пользователями |
| **Менеджер** | Просмотр заявок из формы обратной связи |
| **Редактор** | Управление каталогом товаров и контентными блоками (Главная/Контакты) |

### Управление контентом
- Редактирование блоков на страницах «Главная» и «Контакты» через Markdown-редактор.
- Предпросмотр отрендеренного Markdown перед сохранением.
- Санитизация HTML на бэкенде (защита от XSS).

### Управление каталогом
- CRUD-операции над товарами.
- Поля товара: название, артикул, категория/подкатегория, описание (Markdown), характеристики (key-value), изображение.
- Загрузка изображений: через админку → сохранение в `storage/` → возврат ссылки для вставки в контент.
- Импорт/экспорт данных каталога в форматах: CSV, JSON.

### Заявки с формы связи
- Просмотр списка заявок с фильтрацией по дате/статусу.
- Детальный просмотр: имя, телефон, тема, дата создания.
- Возможность отметить заявку как «обработанную» (опционально).

## 🛡 4. Нефункциональные требования

| Категория | Требование |
|-----------|-----------|
| **Безопасность** | Хэширование паролей, санитизация пользовательского ввода, валидация загружаемых файлов (только изображения, ограничение по размеру), защита от CSRF |
| **Производительность** | Пагинация каталога, ленивая загрузка изображений, оптимизация картинок (WebP/conversion), индексы в БД для поиска |
| **Масштабируемость** | Архитектура позволяет в будущем вынести хранение файлов на S3, добавить кэширование (Redis), вынести бэкенд в отдельный сервис |
| **Расширяемость** | Интерфейс для отправки уведомлений (мессенджеры, email) выделен абстракцией для легкого добавления в будущем |
| **Сопровождаемость** | Логирование ключевых событий, структурированные ошибки в API |
